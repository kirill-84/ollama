# AGENTS.md

Протокол работы AI-агента (Claude Code) с задачами в этом репозитории.
Читать перед каждой задачей.

## Перед началом работы

- Прочитать `CLAUDE.md` целиком.
- Прочитать файлы, упомянутые в issue.
- Если задача противоречит `CLAUDE.md` — не выполнять, оставить
  комментарий в issue с указанием противоречия.

## Работа с ветками и коммитами

- Работать в отдельной ветке `claude/issue-<номер>-<краткий-слаг>`.
  Пример: `claude/issue-7-message-repository`.
- Делать коммиты маленькими, с осмысленными сообщениями в формате
  **Conventional Commits** (`feat:`, `fix:`, `refactor:`, `test:`,
  `chore:`, `docs:`).
- **Никогда не коммитить в `main` напрямую.**

## Тесты

- Писать тест **до или одновременно** с кодом, не после.
- Прогонять `npm test` перед созданием PR.
- Если тест красный — не открывать PR.
- Добавление нового модуля без тестов — повод вернуть задачу
  с комментарием.

## Pull Request

- Создавать PR в `main` с описанием:
    1. Что сделано.
    2. Как проверить.
    3. Ссылка на issue (`Closes #N`).
- В описании — чеклист из acceptance criteria с отметками выполнения.
- Если какой-то критерий не выполнен — явно указать, почему.

## Ограничения на PR

- Максимум файлов в одном PR — **15**. Если больше, разбить на
  подзадачи.
- Не добавлять новые npm-зависимости без явного разрешения в issue.
- Не менять `next.config`, `tsconfig.json`, `eslint.config.mjs`,
  `postcss.config.mjs` без явного разрешения.
- Не менять `prisma/schema.prisma` без явного разрешения — это
  отдельная задача с обязательной миграцией.

## Если что-то неясно

Не гадать и не додумывать. Оставить комментарий в issue с вопросом,
дождаться ответа.

## Формат issue

Каждая задача для агента оформляется по шаблону:

```
feat: <краткое описание>

## Контекст
Что уже есть, какие файлы затрагиваются, какие ограничения.

## Acceptance criteria
- [ ] Конкретный, проверяемый пункт 1
- [ ] Конкретный, проверяемый пункт 2
- [ ] Юнит-тесты покрывают X
- [ ] `npm test` и `npm run build` зелёные

## Что трогать нельзя
- Файл/модуль 1 (причина)
- Файл/модуль 2 (причина)
```
